HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • Elements that are nested inside other elements are called child elements.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.
  • Elements covered in the prework include head, body, footer, h1-6, p, div, a, ul, ol, li, br, main, and img.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • When a developer wants to style an element directly in the HTML file, they use inline CSS.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head
  • Separation of concerns is based on the idea that each section of code should have its own responsibility.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • We can combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.
  • You can use an asterisk to apply a style to all elements, a tag to apply a style to all elements of that tag type, a class to apply a style to elements of that class, or id to apply a style to elements with that id.

Git

  • Git is a tool that allows developers and others to use version control.
  • GitHub is one of many web interfaces for using Git.
  • A repository is a folder in which all files and their version histories are stored.
  • A branch is a version of the repo that allows work without affecting other branches.
  • A fork is a new repository that inherits from a parent “upstream” repo.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git pull origin main updates our local main branch to the latest version from the repo.
  • git add -A stages all adjusted project files on your local computer.
  • git commit -m "" creates a commit to your repo with a message you write in the "".

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is a programming language that allows users to interact with websites. One of its core features is variables, which store data types as values.
  • A semicolon at the end of a line indicates where a statement ends.
  • JavaScript is case sensitive. This means myVariable is not the same as myvariable. If you have problems in your code, check the case!
  • Conditionals are code structures used to test if an expression returns true or not. A very common form of conditionals is the if...else statement.
  • All strings are surrounded by quotation marks.
  • We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
